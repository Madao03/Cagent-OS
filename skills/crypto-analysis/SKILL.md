# Crypto Analysis Skill

> Crypto 市场综合分析框架。对齐三层分析模式方法论（常态/非常态/黑箱），与 macro/us-stock-analysis-skill 同结构。

## 何时触发

当用户消息匹配以下场景时，自动加载本 skill：
- "分析下 XX 项目/代币"、"XX 现在怎么样"、"XX 能买吗"、"XX 为什么涨/跌"
- Crypto 市场环境评估（牛市/熊市/震荡/变盘）
- 资金流向分析（ETF/稳定币/CEX）
- 链上健康度判断（TVL/活跃地址/MVRV）
- 赛道轮动与叙事分析
- 对 crypto 的整体情绪/风险判断
- 涉及杠杆/清算/爆仓的分析

> 若涉及全球宏观分析 → 同时加载 macro-analysis-skill 作为前置输入

## 关联 Skill

| Skill | 关系 | 何时加载 |
|-------|:----:|---------|
| macro-analysis-skill | 上游输入 | 分析全球宏观对 crypto 的影响时，先加载 macro-skill 获得宏观环境结论 |
| us-stock-analysis-skill | 跨市场参考 | 分析 crypto 矿业股（MARA/RIOT）或持币上市公司（MSTR）时加载 |

> 所有数据务必通过工具实时获取。不使用 LLM 记忆中的数字。

调用 CMC MCP `get_global_metrics_latest` 一次性获取：
- BTC 主导率 (BTC.D)
- 总加密市值
- 24h 总成交量
- 恐惧贪婪指数

ETH/BTC 汇率通过 CMC MCP 的 `get_quotes_latest` 分别获取 BTC 和 ETH 价格后手动计算。

### 簇 B：情绪与周期

- 恐惧贪婪指数：CMC MCP `get_global_metrics_latest`（含 fear_greed 字段）
- MVRV-Z Score / 200周MA / AHR999：WebPlugin 搜索 "BTC MVRV-Z score today" / "BTC 200 week moving average"
- 周期位置判断：结合 MVRV-Z + 恐惧贪婪 + 价格 vs 200周MA 综合判断

### 簇 C：链上基础

- 总 TVL / 各链 TVL 排名：WebPlugin 搜索 "DeFiLlama TVL ranking"

### 簇 D：衍生品

- 资金费率 / OI / 多空比：CMC MCP 相关工具，或 WebPlugin 搜索 "BTC funding rate OI today"

### 簇 E：事件与情绪

- 加密新闻/叙事：`FinancialToolkit.search_multi_provider(query="crypto market news today")`
- 财经日历：金十 MCP `list_calendar`

## 步骤 2：常态性分析 — Crypto 结构性关系

> 核心逻辑：识别 crypto 自身内部的结构性关系和周期性规律。

### 2.1 资金流向 ↔ 价格

```text
- ETF 净流入/流出方向 vs BTC 价格走势
- 稳定币总市值变化（增加 → 潜在购买力；减少 → 资金离场）
- CEX 交易所稳定币储备变化
- 判断：资金在流入还是流出？速度在加速还是减速？
```

### 2.2 杠杆与风险结构

```text
- 资金费率：持续正费率 → 多头拥挤；持续负费率 → 空头主导
- 未平仓合约量 (OI)：OI 新高但价格没跟 → 高杠杆风险积累
- 多空比：极端值往往是反向信号
- 基差 (Basis)：正常（年化5-15%）vs 异常（>20%狂热/<0%恐慌）
```

### 2.3 链上健康度

```text
- TVL 总量趋势：上升说明资金在链上活跃
- 活跃地址/DEX交易量：判断链上活动的真实热度
- 各链 TVL 份额变化：资金在 L1/L2 之间流动的偏好
- 赛道 TVL 占比变化：DeFi / LSD / RWA / Meme 等赛道份额轮动
```

### 2.4 周期位置判断

| 指标 | 熊市底部 | 早期牛市 | 狂热期 | 泡沫 |
|:----|:-------:|:--------:|:-----:|:---:|
| MVRV-Z | < 0 | 1-3 | 3-5 | > 5 |
| 恐慌贪婪 | < 20 | 30-60 | 70-85 | > 90 |
| AHR999 | < 0.5 | 0.8-1.5 | 2-5 | > 5 |
| 价格 vs 200周MA | 破位 | 站上 | 远高于 | 极端偏离 |
| 资金费率 | 负/近零 | 正常正 | 高正 | 极高 |
| BTC.D | 高(>60%) | 下降 | 低位 | 极低 |

### 2.5 代币级分析 — 生态繁荣 ≠ 代币捕获

分析单个非BTC/ETH代币时的额外检查点：

```text
- 代币在生态中扮演什么角色？gas / 治理 / 质押 / 激励 / 手续费分成？
- 用户是否必须持有代币才能使用链上服务？（Gas能用USDT付吗？）
- TVL / DAU / 用户数 → 哪个指标与代币价格有实质关联？
  - 高DAU不一定利好代币（例：用户用稳定币转账，不需要原生代币）
  - 重点看 TVL 增长 + 代币捕获机制变化
- DEX流动性是否足够？低流动性 → 持仓风险高，退出困难
- 主要上币交易所覆盖？仅靠一个流动性池是否足够
```

### 2.6 代币经济模型变更分析（Buyback/Burn/Supply 变更）

#### 数据收集

```text
1. 获取代币供给数据：total_supply / circulating_supply / max_supply
   → CMC MCP 或 WebPlugin 搜索
   - 计算未流通占比 = (total - circulating) / total
   - 确认变更前已销毁量、已回购未销毁量

2. 获取协议收入数据：
   - 月均净收入估算（WebPlugin 搜索 DeFiLlama / Dune 仪表板）
   - 变更后每月销毁量 = 月净收入 × 新销毁比例 ÷ 当前代币价格
     → 月销毁率 = 月销毁量 / 流通总量
     → 年化销毁率 = 月销毁率 × 12

3. 获取解锁/通胀数据（对冲判断）：
   - 团队/投资人/国库持有量 = total_supply - circulating_supply
   - 是否有线性解锁计划？解锁速率 vs 销毁速率的净差额
   - 净供应变化率 = 月解锁量 - 月销毁量
```

#### 「死钱 vs 活钱」框架

```text
Before 变更：收入 → 回购 → 国库锁仓（死钱）
  - 代币被"冻住"，既不对供应产生影响，也不能被有效使用
  - 结果：什么也没发生（零通缩、零生态投入、零激励对齐）

After 变更：收入 → 部分销毁 + 部分运营投入（活钱）
  - 销毁部分 → 直接通缩，代币持有者直接受益
  - 运营部分 → 团队有资金搞发展，有动力产生更多收入
  - 结果：三方受益（持币者获得通缩、团队获得弹药、社区获得发展）
```

#### 净供应变化率公式

```text
净供应变化率（月）= 月解锁量（枚） - 月销毁量（枚）
净供应变化率（%）= 净供应变化率 / 流通总量

判断：
  - 净供应变化率 > 0 → 通胀仍在加速，币价承压
  - 净供应变化率 ≈ 0 → 销毁刚好抵消解锁，币价中性
  - 净供应变化率 < 0 → 硬通缩启动，对币价有结构性支撑

⚠️ 常见误区：只看销毁量而忽略未流通代币的解锁计划。
如果未流通占比 > 40-50%，即使有销毁也可能只是因为解锁速度更快。
```

#### 估值比率定位

```text
Mcap / TVL (Mcap-to-TVL ratio):
  - <1x: 严重低估
  - 1-3x: 合理偏低
  - 3-5x: 板块均值
  - >10x: 叙事溢价驱动

FDV / TVL: 比 Mcap/TVL 更能反映真正估值（含未解锁代币）
```

## 步骤 3：非常态性分析 — 事件冲击与传导

> 核心逻辑：事件不直接冲击 crypto 价格，而是通过特定的传导链。

### 3.1 监管/合规事件传导

```text
传导链：监管动作 → 具体项目/交易所受限 → 市场情绪承压 → 资金避险流出
示例：SEC起诉Binance → CEX流出 → OI暴跌 → 价格下行
```

### 3.2 安全事件/黑客攻击传导

```text
传导链：合约漏洞/跨链桥被黑 → TVL流失 → 链上信任度下降 → 相关代币抛售 → 蔓延恐慌
评估：损失金额 / 影响协议数 / 是否有保险覆盖
```

### 3.3 宏观传导（通过 macro-analysis-skill 输入）

```text
macro-skill 输出 → 宏观环境评级（利好/中性/利空风险资产）
     ↓
结合 crypto 当前自身的杠杆/资金结构判断：
- 若 crypto 本身杠杆已高 + 宏观利空 → 清算级联风险
- 若 crypto 去杠杆已充分 + 宏观转向 → 反转启动机会
```

### 3.4 黑天鹅场景

```text
- 交易所重大事件（FTX式）
- 稳定币脱锚（UST式）
- 核心基础设施严重故障
处理：确认事件 → 评估影响面 → 输出恐慌/机会判断
```

## 步骤 4：黑箱观察 — Crypto 内部市场联动

> 核心逻辑：不分析过程，只看结果之间相互联动。

### 4.1 BTC ↔ ETH ↔ Altcoin

```text
- BTC.D 上升：资金集中在 BTC，altcoin 弱势（避险/熊市）
- BTC.D 下降：资金流向 altcoin（风险偏好上升/牛市）
- ETH/BTC 汇率：ETH 相对 BTC 走强 → 市场信心回升
- Top 10 altcoin 是否跟涨/跟跌 BTC
```

### 4.2 衍生品信号

```text
- 资金费率 + OI 组合判断：
  ┌─────────────┬──────────────┬─────────────────┐
  │ 资金费率     │ OI           │ 信号             │
  ├─────────────┼──────────────┼─────────────────┤
  │ 持续正       │ 持续上升      │ 看多但拥挤       │
  │ 持续负       │ 持续上升      │ 看空但可能爆空   │
  │ 突然负       │ 骤降          │ 恐慌平仓（暴跌） │
  │ 正常/零      │ 下降          │ 杠杆出清（健康） │
  └─────────────┴──────────────┴─────────────────┘
```

### 4.3 CEX ↔ DEX 联动

```text
- CEX 深度 vs DEX 交易量：判断流动性在哪
- Coinbase 溢价指数 > 0：美国投资者正溢价买入
- CME 期货基差：机构 vs 散户的定价偏差
```

### macro-skill 联动输入格式

若先跑了 macro-analysis-skill，以下宏观变量可直接引用：

| 宏观变量 | 对 crypto 的传导含义 |
|:---------|:-------------------|
| 利率走势（降息/加息） | 降息→流动性宽松→利好 crypto，加息→紧缩→利空 |

| 失业率 | 失业率上升→经济放缓预期→加速降息预期→利好 crypto |
| 政策匹配度 | 评级：利好/中性/利空 — 影响 crypto 的宏观贝塔方向 |
```

## 注意事项（Pitfalls）

- BTC.D 上升不等于 bearish——要看上升是因为 BTC 涨还是 altcoin 跌
- 不要跳过传导链——「地缘战争 → BTC 跌」太跳跃，必须写出完整传导路径
- 用户活跃度 ≠ 代币价值捕获 — 高DAU不一定利好代币，必须先问：代币在这个生态中到底起什么经济作用？
- 不要因为一个合作/新闻就说基本面改善 — 必须量化：合作带来了多少增量TVL/用户/营收？
- 数据不可获取时标 `[数据不可实时获取]`，不要编造——用已有数据继续分析
- 所有数据标注来源和时间
- 预测性结论标注置信度：高 / 中 / 低
- macro-skill 和 crypto-skill 的输出可交叉验证——宏观说利空但 crypto 资金在流入 → 可能有分歧
