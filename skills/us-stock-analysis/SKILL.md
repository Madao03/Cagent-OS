# US Stock Analysis Skill

> 美股个股分析框架。对齐三层分析模式方法论（常态/非常态/黑箱），与 macro/crypto-analysis-skill 同结构。
> 数据主力：FinancialToolkit（yfinance + fin-skill MCP 双源交叉验证）。

## 何时触发

当用户涉及以下场景时自动加载：
- 分析某只美股个股的基本面/估值/前景
- 财报季个股点评
- 个股买入/卖出/持有建议
- 个股内推报告生成（initiating coverage 简化版）
- 行业板块内个股比较
- 涉及宏观环境判断 → 同时加载 macro-analysis-skill 作为前置输入
- 涉及 crypto 个股（如 MSTR）→ 同时加载 crypto-analysis-skill

## 数据源

| 工具 | 用途 | 覆盖 |

## 步骤 1：数据收集（全量并行）

> 所有数据务必通过工具实时获取。不使用 LLM 记忆中的数字。

### 簇 1：公司基本信息 + 实时行情

调用 `FinancialToolkit.query_quote(ticker)` 获取：
- 公司全称、行业分类、交易所、业务描述
- 当前价、前收盘、开盘价、日高/日低、成交量
- 52周高/低、Beta
- 市值、企业价值 EV

### 簇 2：估值指标

从 `query_quote` 返回中提取：
- PE (TTM)、Forward PE、PB、PS
- EV/Revenue、EV/EBITDA、PEG

### 簇 3：盈利能力

从 `query_quote` 返回中提取：
- EPS (TTM)、Forward EPS
- 毛利率、营业利润率、净利润率
- ROE、ROA
- 营收增长、盈利增长
- 营收 (TTM)、EBITDA

### 簇 4：财务健康

从 `query_quote` 返回中提取：
- 总现金、每股现金、总负债
- 负债/权益、流动比率、速动比率
- 自由现金流、经营现金流

### 簇 5：股东回报

从 `query_quote` 返回中提取：
- 股息率、派息率、5年平均股息率
- 回购/股票数

> 无股息公司（如 AMZN/GOOGL）：dividendYield 为 None 或 0，标注"N/A"，不要误判为股息率 0%。

### 簇 6：持股结构与做空

从 `query_quote` 返回中提取：
- 机构持股比、内部持股比、流通股
- 做空比、做空天数

### 簇 7：分析师评级

从 `query_quote` 返回中提取：
- 目标价均值/低/高/中位数
- 推荐等级（buy/strong_buy/hold/sell）
- 推荐均值（1=强力买入 ~ 5=卖出）
- 分析师数

### 簇 8：完整财务报表

调用 `FinancialToolkit.query_earnings(ticker)` 获取：
- 利润表：Total Revenue、Gross Profit、Operating Income、Net Income、EBITDA、Diluted EPS
- 资产负债表：Total Assets、Total Liabilities、Stockholders Equity、Cash、Long Term Debt
- 现金流：Free Cash Flow、Operating Cash Flow、Capital Expenditure

### 簇 9：新闻与催化剂

调用 `FinancialToolkit.search_multi_provider(query="{ticker} earnings news catalyst")` 获取：
- 最新财报解读
- 分析师评级调整
- 公司重大事件

### 簇 10：行业与宏观背景

- `WebPlugin` 搜索行业对比数据
- 加载 `macro-analysis-skill` 获取宏观环境判断

## 步骤 2：常态性分析 — 估值与质量评估

> 核心逻辑：判断股票当前是否处于合理的估值区间，以及公司基本面质量。

### 2.1 估值历史分位（PE/PS/PB vs 历史）

```text
- PE TTM vs 公司5年历史PE区间：处于____分位（偏高/适中/偏低）
- Forward PE vs TTM PE：折溢价____%（Forward PE < TTM PE → 盈利增长预期）
- PEG < 1 → 可能被低估；PEG > 2 → 增长已被定价
- EV/EBITDA vs 行业：____（溢价/折价____%）
- PB 判断：轻资产（科技）PB > 3 正常；重资产（制造/银行）PB < 2 正常
```

### 2.1.1 周期股估值陷阱（必须先看这个）

> 如果 Forward PE < 10x，先假设是陷阱，再去找反证。

周期股（半导体制造/内存/化工/航运/钢铁）在周期顶部时，Forward PE 看起来极便宜——但这是高点的幻觉。

**陷阱机制：**

```
周期顶部的经典画像：
  Forward PE < 10x       ← 看起来便宜
  PEG < 0.5              ← 看起来极度便宜
  + 52 周涨幅 > 300%     ← 但你已经在山顶了
  + 毛利率创历史新高     ← 均值回归在路上了
  - CAPEX/OCF > 60%      ← 利润正在被设备供应商抽走
  - Forward EPS 60-90 天内翻倍 ← 分析师线性外推暴利
  - 分析师一致看多       ← 预期已经在云端了

为什么 Forward PE 在周期顶部会失真：
  Forward PE 使用的 EPS 是「分析师对未来 12 个月的预测」
  → 在周期顶部，分析师把当前超高的盈利线性外推
  → FY22 MU Forward PE = 5x（当时看起来极便宜）
  → FY23 EPS 转负 → PE 变成无穷大 → 股价腰斩
  → 不是 PE 骗了你，是你不知道当前盈利不可持续
```

**快速检查清单：**

```
□ 52 周涨幅 > 200%？→ 如果 Yes，暂停用 PE/PEG 做估值判断
□ 毛利率超过 10 年均值 + 2 个标准差？
□ CAPEX/OCF > 60%？（如果是，利润质量存疑）
□ 公司是否在历史上经历过 EPS 转负？
□ Forward EPS 60-90 天内暴涨 > 50%？

如果以上 ≥ 2 个 Yes：
  → 大概率处于周期顶部区域
  → 切换到 PB 周期分位法 + EV/EBITDA（跨周期可比）
  → Fwd PE 和 PEG 的数字是噪音，不是信号

何时 PE/PEG 恢复正常意义：
  → 周期底部：52 周跌幅 > 50% + CAPEX 开始削减 + 毛利率跌破均值
  → 此时 PE 可能极高（EPS 已崩），但 PB 处于历史低位
  → 这才是在周期股上使用 PE 的正确时机——因为 PE 的「高」反映的是盈利底部
```

**周期股正确估值工具对照：**

```
周期位置      | 该用什么          | 不该用什么
──────────────|─────────────────|─────────────
周期顶部       | PB 分位法        | PE / PEG / DCF
              | EV/EBITDA       |
周期中部       | EV/EBITDA       | PEG（仍可能失真）
              | PB 分位法        |
周期底部       | PE（因为盈利低）  | PEG（分母过小）
              | PB 分位法        |
              | EV/EBITDA       |
```

### 2.1.2 产品换代透支风险（Pull-Forward Risk）

> 当一家公司刚完成重大产品换代（新架构GPU、新iPhone、新车型），先检查Q1是否透支了Q2的需求。

**陷阱机制：**

```
产品换代透支的经典画像：
  + 全新产品架构刚发布/量产（Blackwell、iPhone 18、Cybertruck）
  + Q1 营收 QoQ 增长 > 15%（远超正常季节性）
  + 客户在电话会上提到「战略性囤货」或「首批部署」
  → Q1 的高增速可能部分是客户囤货，不是可持续的消耗需求
  → Q2 如果环比下降，市场叙事会瞬间切换：
    「超级周期加速」→「一次性囤货脉冲」
  → 触发 EPS 下修 + PE 压缩 = 双杀
```

**快速检查清单：**

```
□ 最近一个季度是否有重大新产品发布/量产？
□ QoQ 营收增速是否 > 15%（大基数下更敏感）？
□ 电话会中是否出现「initial deployment」「building inventory」等表述？
□ 下游客户（云厂商/经销商）的存货水平是否在上升？

如果以上 ≥ 2 个 Yes：
  → 存在产品换代透支风险
  → 不要用 Q1 年化来估计全年 EPS —— 共识可能已经高于 Q1 年化
  → 核心跟踪指标：Q2 环比是否不降（≥Q1 = 透支被证伪，<Q1 = 透支被确认）
```

> 产品换代透支 ≠ 周期顶部——它不是结构性问题，而是时序错配。区分「生意本身好不好」和「市场当下怎么交易它」。

### 2.2 增长质量

```text
- 营收增长：过去3-5年 CAGR vs 最近的增速变化（加速/减速/稳定）
- 利润增长 vs 营收增长：利润增长是否快于营收→运营杠杆正在释放
- 利润率趋势：毛利率/营业利润率/净利率过去3年趋势（扩张/压缩/稳定）
- FCF 转化率：FCF / Net Income — >80% 优秀，<50% 需关注
```

### 2.3 竞争优势（Moat）评估

| 维度 | 强 | 中 | 弱 |
|:----|:--|:--|:--|
| 网络效应 | 用户基数越大越有价值 | 有一定网络效应 | 无网络效应 |
| 转换成本 | 客户更换成本极高 | 有一定粘性 | 低转换成本 |
| 规模经济 | 大规模带来成本优势 | 有规模但不明显 | 无规模优势 |
| 无形资产 | 品牌/专利/监管壁垒强 | 有一定壁垒 | 无壁垒 |
| 数据/算法飞轮 | 使用越多数据越强 | 有数据积累 | 无数据优势 |

### 2.4 财务健康评分

```text
- 流动性：current ratio > 1.5 ✅ / 1.0-1.5 ⚠️ / < 1.0 🔴
- 杠杆：D/E < 50% ✅ / 50-100% ⚠️ / > 100% 🔴（行业标准有差异）
- 利息覆盖：EBIT / Interest > 5x ✅ / 2-5x ⚠️ / < 2x 🔴
- 现金充足度：cash / debt > 1.0 ✅ / 0.5-1.0 ⚠️ / < 0.5 🔴
```

## 步骤 3：非常态性分析 — 催化剂与风险事件

> 核心逻辑：识别近期的催化剂和风险事件，判断对股价的潜在影响方向与幅度。

### 3.1 财报催化剂

```text
- 下次财报日：____（来自金十 MCP list_calendar）
- 共识预期：EPS $____ / Revenue $____（来自 WebPlugin 搜索最新共识）
- 意外历史：过去4个季度的 beat/miss 记录（来自 query_earnings）
- 估值隐含预期：当前股价隐含了怎样的增长预期？
```

### 3.2 公司催化剂

```text
- 新产品/服务发布
- 重大客户/合同签订
- 管理层变动
- 资本配置变化（回购加速/派息增加/并购）
- 分拆/剥离/上市
- 监管/法律裁决（FDA 审批/反垄断/诉讼）
```

### 3.3 公司特有风险

```text
- 客户集中度：单一客户占比？
- 技术颠覆风险：是否有替代技术威胁？
- 竞争加剧：市场份额是否被侵蚀？
- 监管风险：行业是否面临政策变化？
- ESG/声誉风险
- ADR 特有风险：中概/台股 ADR 的退市风险、审计底稿争议
```

### 3.4 行业与宏观敏感性

```text
- 行业周期位置：上行/下行/结构性变化
- 利率敏感度：高（金融/地产）/ 中（科技/消费）/ 低（必需消费/公用）
- 经济敏感度：强周期（工业/材料）/ 弱周期（医疗/消费必需品）
- 商品/汇率敞口：原材料成本/海外收入占比 → 汇率风险
```

### 3.5 做空/估值争议

```text
- 做空比例 > 5% + 做空天数 > 10 → 挤压风险高
- 做空报告/Grizzly/Citron/浑水 最近是否盯上？
- 公司是否有重大财务争议？（应收账款激增、资本化费用异常等）
```

## 步骤 4：黑箱观察 — 价格行为与市场信号

> 核心逻辑：不分析原因，只观察价格行为和市场信号之间的联动。

### 4.1 技术趋势

```text
- 当前位置 vs 50日MA vs 200日MA：
  ┌──────────────┬─────────────────────────┐
  │ 模式          │ 信号                     │
  ├──────────────┼─────────────────────────┤
  │ 价格 > MA50 > MA200 │ 多头排列（趋势向上）  │
  │ MA50 > 价格 > MA200   │ 短期回调（中期仍好）│
  │ MA50 < MA200 < 价格   │ 初步反转           │
  │ 价格 < MA50 < MA200   │ 空头排列（趋势向下）│
  └──────────────┴─────────────────────────┘
- 成交量确认：放量突破 ✅ / 缩量上涨 ⚠️ / 放量下跌 🔴
- 近期是否有缺口：gap up/down + 缺口回补情况
```

### 4.2 相对表现

```text
- vs SPX：年初至今相对收益 ____%
- vs 行业板块ETF（XLK/XLY/XLI等）：____
- vs 直接竞争对手：____
- 强势/弱势是否在持续或逆转
```

### 4.3 期权市场信号

```text
- 隐含波动率（IV）百分位：偏高 → 期权定价反映不确定性
- 隐含波动率（IV）vs 历史波动率（HV）：IV > HV → 市场预期波动加大
- 期权隐含涨跌（ATM straddle）：市场预期财报日波动 +-____%
- Put/Call 比率（如果可获取）
```

### 4.4 内部人交易信号

```text
- 近期是否有重要的内部人买入/卖出（通过 WebPlugin 获取）
- insider 卖出 ≠ 利空（可能只是 diversification）
- insider 买入 > $1M → 通常为看好信号
```

## 数据质量注意事项

### Forward PE / EPS 数据质量

- `DataWall` 自动交叉验证 YFinanceAdapter 和 FinSkillAdapter 的 PE 数据
- 不同数据源可能使用不同的底层供应商，Forward PE 可能存在显著差异
- 单一源不可用时，交叉验证器自动回退到可用源，置信度 0.5

### 工具使用原则

⚠️ **数据检索优先级: RAG > Web Search** — 先查本地知识库（已归档 29 篇研报/新闻/台账），命中不到再搜外部。

- **`financial.rag.search`** — 优先调用的知识库检索，查已归档的美股分析/新闻
- 每次分析必须通过工具实时获取数据，不使用 LLM 记忆中的数字
- `FinancialToolkit.query_quote` 是首选入口（28 个指标一站式获取）
- `FinancialToolkit.query_earnings` 补充深度财报数据
- `WebPlugin` 用于搜索引擎无法覆盖的实时信息
- 金十 MCP 日历用于财报日确认
- **`financial.websearch`** — 强制检索对立观点：搜索 "{标的} bull case" + "{标的} bear case"，呈现至少一个有明确来源的反对意见
