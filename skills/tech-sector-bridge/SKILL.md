# Tech Sector Bridge — 宏观→科技板块传导

> **定位**：macro-analysis 和 us-stock-analysis 之间的桥梁。
> 不替代任何一个——通常同时加载 macro-analysis（获取宏观输入），然后输出板块判断给后续的 us-stock-analysis 使用。

## 何时触发

- 用户问科技板块整体方向（「科技股现在该超配吗」「半导体还能不能买」）
- 用户问板块轮动（「现在资金在往哪条赛道转」）
- 用户做多股比较（「NVDA 和 MSFT 哪个更好」→ 先定板块基调，再切 us-stock-analysis）
- 用户问某个宏观事件对科技股的影响（「降息对科技股利好吗」→ 直接答）
- ⚠️ 加载本 Skill 时，通常同时加载 **macro-analysis**（获取宏观输入）

## 关联 Skill

| Skill | 关系 | 加载时机 |
|:------|:----:|:--------|
| **macro-analysis** | 上游输入 | 同时加载，获取利率/流动性/增长预期的完整判断 |
| **us-stock-analysis** | 下游消费者 | 本 Skill 输出板块判断后，切 us-stock-analysis 做个股深挖 |
| **crypto-stock-analysis** | 互补 | 币股有独立的估值逻辑，不适用传统科技板块框架 |

## 步骤 1：获取宏观输入

> 如果已经加载 macro-analysis 并产出了宏观快照，跳过此步骤直接引用。

快速获取核心变量：
- 利率：`financial.websearch("US 10Y Treasury yield")`
- 流动性：`financial.websearch("Federal Funds rate")` 或 `financial.quote.verified`
- 市场情绪：`financial.quote.query` 获取 SPX/VIX

输出宏观快照：
```
宏观环境：10Y=X% | 利差=±X% | FEDFUNDS=X% | VIX=X | SPX=方向
→ 对科技股：🟢顺风 / 🟡中性 / 🔴逆风
```

## 步骤 2：宏观 → 科技板块传导

> 核心逻辑：**同一组宏观变量，对不同科技子赛道的影响方向和幅度不同。**

### 2.1 利率传导矩阵

| 利率变动 | 半导体 | 软件/SaaS | 互联网/广告 | 消费硬件 | 传导逻辑 |
|:--------|:------:|:---------:|:----------:|:-------:|:--------|
| **利率↑**（收紧） | 🔴 中度利空 | 🔴🔴 强利空 | 🟡 弱利空 | 🔴 中度利空 | DCF 分母↑，高增长远期现金流被惩罚最重 |
| **利率↓**（宽松） | 🟢 强利好 | 🟢🟢 最强利好 | 🟢 利好 | 🟢 利好 | 软件股 duration 最长，降息受益最大 |
| **利差倒挂** | 🔴🔴 强利空 | 🔴🔴 强利空 | 🔴 利空 | 🔴 利空 | 衰退预期 → 企业 IT 预算砍 → SaaS 首当其冲 |
| **利差转正** | 🟢🟢 最强利好 | 🟢 利好 | 🟢 利好 | 🟢 利好 | 软着陆 → 半导体 AI Capex 叙事不会被衰退打断 |

### 2.2 流动性传导矩阵

| 流动性变化 | 半导体 | 软件/SaaS | 互联网/广告 | 消费硬件 |
|:----------|:------:|:---------:|:----------:|:-------:|
| **QT 缩表** | 🔴 利空 | 🔴 利空 | 🟡 中性 | 🔴 利空 |
| **降息+结束QT** | 🟢 强利好 | 🟢 强利好 | 🟢 利好 | 🟢 利好 |
| **信贷收紧** | 🔴🔴 强利空 | 🔴 利空 | 🟡 中性 | 🟡 中性 |

### 2.3 增长预期传导

| 增长信号 | 半导体 | 软件/SaaS | 互联网/广告 | 消费硬件 |
|:--------|:------:|:---------:|:----------:|:-------:|
| **GDPNow > 预期** | 🟢🟢 | 🟢 | 🟢🟢 | 🟢 |
| **ISM PMI > 50** | 🟢🟢 最强利好 | 🟢 | 🟢 | 🟢 |
| **ISM PMI < 45** | 🔴🔴🔴 危险 | 🔴🔴 | 🔴🔴 | 🔴🔴 |

输出传导判断：
```
当前宏观组合对科技板块：
  利率信号：[利空/中性/利好]
  流动性信号：[利空/中性/利好]
  增长信号：[利空/中性/利好]

综合传导：半导体[🟢🟡🔴] 软件[🟢🟡🔴] 互联网[🟢🟡🔴] 消费硬件[🟢🟡🔴]
```

## 步骤 3：子赛道基本面定位

### 3.1 半导体 — 周期位置

```
领先指标：台积电月度营收 YoY / 云厂商 Capex 指引 / SEMI Billings
当前周期位置：[扩张初期/繁荣期/顶部/下行]
核心问题：AI Capex 的持续性
```

### 3.2 软件/SaaS — 增长质量

```
关键指标：MSFT/CRM/ADBE 的 ARR 增速、净 retention rate、Fwd PE vs 历史分位
当前状态：[估值合理+增速稳定/估值高+增速放缓/估值低+增速触底]
核心问题：企业 IT 预算是否从「实验 AI」转向「部署 AI」
```

### 3.3 互联网/广告 — 变现效率

```
关键指标：META/GOOGL 广告收入增速、AI 对广告精准度的提升
核心问题：AI 对广告 ROI 提升是否已 price in
```

### 3.4 产业结构健康度 — 「这是一门好生意吗？」

> 借鉴「航空公司类比」框架：有些生意的结构性缺陷不会因为 AI 标签消失。

**四个维度评分：**

```
维度 1：ROIC vs WACC（长期价值创造）
  ROIC > WACC + 5% 持续 10 年 → 🟢 有护城河
  ROIC ≈ WACC ± 2%           → 🟡 勉强维持
  ROIC < WACC 持续 5 年以上    → 🔴 结构性差生意
  ⚠️ ROE 高 ≠ ROIC 高——借钱能推高 ROE，ROIC 暴露真相

维度 2：CAPEX 结构（谁在拿走利润）
  CAPEX/OCF < 30%  → 🟢 轻资产，利润属于股东
  CAPEX/OCF 30-60% → 🟡 中等负担
  CAPEX/OCF > 60%  → 🔴 利润被设备供应商抽走

维度 3：定价模式（谁决定价格）
  技术溢价定价 → 🟢 公司掌握定价权（NVDA GPU）
  寡头定价     → 🟡 几家共同维持（台积电先进制程）
  大宗商品定价 → 🔴 供需决定价格（DRAM/NAND 内存）

维度 4：周期长度与当前位置
  半导体：5-7 年完整周期
  内存（DRAM/NAND）：3-4 年，更短更烈
  领先指标：52周涨幅 >300%？/ 毛利率历史新高？/ CAPEX 仍在加速？
```
